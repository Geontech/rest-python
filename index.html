REDHAWK REST API

This project is a fork of the REDHAWK SDR implementation with several tweaks and extensions added to support UI development. However like that project, ours has also lacked any detailed documentation for the API the server provides. So in addition to our Wiki pages, please use this site as your guide to developing web-based (or otherwise) apps.

General Concepts

At its core, this server taps into the REDHAWK SDR python sandbox. Many of the activities you can do in that environment also apply to this server, but are wrapped with two different sets of interfaces.

The first are HTTP requests including GET, POST, etc. These are typically for getting and setting static elements from the REDHAWK system like Component names or properties.

The second interface is through web sockets. These are exclusively for asynchronous interactions such as event channel or BULKIO transfers. At this time, these transfers are unidirectional from the server up to your client.
